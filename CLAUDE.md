# eth-protocol-fellowship, Agent Notes

Public site for Ivan Anishchuk's Ethereum Protocol Fellowship (EPF) cohort-seven
dev updates and project updates. Built with zensical, deployed to GitHub Pages at
`https://ivananishchuk.github.io/eth-protocol-fellowship/`. No custom domain, so
the archive keeps a stable URL for the long term.

See `PLAN.md` for the staged build plan and acceptance criteria.

## What this is

A documentation-style hub plus an "Updates" blog. The hub frames the etheorem
formal-verification work and links the project. The blog carries the biweekly EPF
dev updates, with two further categories for project updates and news. The
content outlives the fellowship as a public record.

## Hard rules

- **Dev-update prose is self-written (no-LLM).** EPF requires every dev update to
  be the fellow's own writing. An agent may scaffold a post (front matter,
  headings, a links checklist) and may review or copy-edit a finished draft. An
  agent must never write the body prose of a weekly dev update. The hub, about,
  and project pages may be agent-drafted, then Ivan rewrites them in his own
  voice.
- **No third-party artwork gets committed.** Ethereum.org and Ethereum-blog
  visuals are style inspiration for palette and layout only. Do not download,
  commit, or redistribute Ethereum Foundation illustrations. Per-page social
  cards are generated by our own build step from the theme colors, our own logo,
  and the page title.
- **Public repo hygiene.** No secrets, no large media, no build output. The
  generated `site/` directory and `docs/.bloggen/` stay untracked.

## Stack

- **zensical**, the static site generator from the Material for MkDocs team, and
  Ivan's house pattern (`cuere`, `oauth`, `nozkash/docs` all use it). Config lives
  in `zensical.toml`. Build with `uv run --no-sync zensical build --clean
  --strict`, output to `site/`. Copy the theme, palette, features, and
  markdown-extension block from `~/src/IvanAnishchuk/cuere/zensical.toml`.
- zensical does not yet ship a blog plugin or social-card generation. We add both
  ourselves. This is the accepted cost of staying on zensical, decided by Ivan
  over switching renderers. Track upstream and contribute back where it fits.
  - **Blog:** the `knu2xs/zensical-blog` plugin, pinned to a commit. Vendor it
    into the repo if we need to patch it. If it does not integrate cleanly, fall
    back to a hand-rolled dated-posts structure (the posts are plain markdown
    either way, so content survives a plugin swap).
  - **Social cards:** a custom generation step (for example a small Python script
    using Pillow or cairosvg, or a port of Material's social-card logic) that
    emits a per-page Open Graph image and injects the `og:image` and
    `twitter:image` meta.
- **uv** for the Python environment. Pin versions in `pyproject.toml`; commit
  `uv.lock`.
- **CI:** GitHub Actions builds and deploys to Pages. Adapt the hardened workflow
  at `~/src/IvanAnishchuk/cuere/.github/workflows/docs.yml` (pinned action SHAs,
  harden-runner, `persist-credentials: false`, least-privilege permissions). It
  already runs `uv run --no-sync zensical build --clean --strict` and deploys
  `site/`. Add a step for the social-card generator and any system libraries it
  needs.

## Conventions

- Blog posts live at `docs/blog/posts/YYYY-MM-DD-slug.md` with `date` and
  `categories` in the front matter. Allowed categories: `weekly-update`,
  `project-update`, `news`.
- Pin GitHub Actions to commit SHAs. Re-verify the pins when CI lands, since they
  go stale.
- Run `uv run zensical serve` for local preview; `uv run zensical build --clean
  --strict` is the gate.

## Writing style

These mirror Ivan's global conventions and bind on docs, comments, commit
messages, and PR text.

- No em-dashes for subphrases. Use commas, or split into two sentences.
- No contrastive negation ("not X, but Y"). State the positive directly.
- Vary sentence length. Prefer active voice.
- No filler openers ("Let's dive in") or summary closers ("In conclusion").
- Avoid AI-tell vocabulary (delve, leverage, robust, seamless, tapestry,
  landscape, and similar).
