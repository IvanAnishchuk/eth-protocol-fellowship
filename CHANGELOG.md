# Changelog

All notable changes to this project are documented here. The format follows
[Keep a Changelog](https://keepachangelog.com/en/1.1.0/). This site deploys
continuously, so entries accumulate under [Unreleased].

## [Unreleased]

### Added

- uv build environment: `pyproject.toml` and `uv.lock` pinning zensical
  (`>=0.0.45,<0.1`). The scaffold builds with
  `uv run zensical build --clean --strict`.
- Minimal `zensical.toml` and a placeholder home page.
- Updates section: a static stub page at `docs/updates/index.md`, linked from
  the nav. The dated dev-update log will be generated by a separate
  `zensical-updates` sidecar (a clean-room CC0 rewrite, own repo) once it ships.
- Theme and branding: the indigo/violet identity over zensical's modern theme.
  A custom palette (`primary`/`accent` = `custom`) with a light and dark scheme,
  the brand stylesheet `docs/stylesheets/brand.css`, the logo and favicon, and
  the standard Material markdown-extension set. The dark scheme re-tints to the
  brand ink (`--md-hue: 244deg`); the gradient appears as the page-title
  wordmark and thin header/footer rules. Typography is Hanken Grotesk (body),
  Archivo (display headings), and JetBrains Mono (code).
- SEO base: per-page meta descriptions and canonical URLs from `site_url`;
  zensical emits `sitemap.xml` natively, so no extra build step is needed.
- Hub pages: `docs/index.md` (home), `docs/project.md` (the formal-verification
  project), and `docs/about.md` (bio, fellowship context, contact). Each carries
  a `description` front-matter line for its meta tag. The nav now reads Home,
  Project, Updates, About. Agent-drafted; Ivan rewrites the prose in his voice.

### Changed

- `docs/assets/logo.svg` now uses the brand gradient (was white-only), so the
  mark stays visible on both the light and dark headers.

### Notes

- The `zensical-blog` plugin spike concluded that a single-build plugin is not
  yet possible (zensical 0.0.45 does not discover `zensical.plugins` entry points
  or call plugin hooks), and the upstream sidecar mutates committed files and
  emits post links that do not match where zensical renders them. We will build
  our own generator instead, so the plugin dependency is dropped for now.
